# Security Policy

## Reporting a vulnerability

If you think you've found a security issue in highlighters, please report it privately. Do **not** open a public GitHub issue for vulnerabilities.

You can report in either of these ways:

- Open a private advisory via [GitHub Security Advisories](https://github.com/JaceThings/highlighters/security/advisories/new) (preferred).
- Email hi@ja.mt.

Please include enough detail for the issue to be reproduced: affected package and version, a minimum reproduction, and the impact you've observed.

## Response expectations

- Initial response: within **48 hours** on a best-effort basis.
- We'll confirm the issue, agree a disclosure timeline with you, and ship a fix plus advisory once a patch is ready.
- Credit is given in the advisory unless you prefer to stay anonymous.

## Supported versions

highlighters is in `0.x`. Only the latest published version is supported with security fixes.

| Version | Supported |
| ------- | --------- |
| latest  | Yes       |
| older   | No        |

Once highlighters hits `1.0`, this table will expand to cover supported major lines.
